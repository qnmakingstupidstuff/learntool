I will explain how the Encryption Method works.

The first type of RSGP or PGSR is called "Uncompressed". This type of Format is Popular since 1.4.2 to 3.1 but after that PopCap & EA switched into the type "Compressed". This isn't a popular type of RSGP anymore but they are still using it. One common is "StreamingWave" and "__MANIFESTGROUP__". PvZ2 Chinese still use this type of RSGP, however.

The second type of RSGP or PGSR is called "Compressed". This is the most popular structure in the RSB or OBB. If "__MANIFESTGROUP__" from "Uncompressed" to "Compressed", the game will still run normally but not include for "StreamingWave".

Infact, Encryption is very very easy. They just used the "bug" called "corruption". They will "corrupted" the tools which read header of the rsgp and see the "01" or "03" to handle the RSGP. So, they just change the number "01" or "03" to something similar to "00" or "FF"

The next type of Encryption is the RSB Read Encryption. Although this Encryption method is old, people still like it and use it. Take example from the SHUTTLE OBB by CZ

So with this encryption, tool like Taiji, SPC-Util or TwinKleS when read will jump into the Offset "FFFFFFFF" and found nothing because that Offset doesn't existed. In order to do this kind of encryption, make a blank RSGP with anything and change the offset to "FFFFFFFFF". With this way, you are encrypting the OBB from powerful tool. However, i will teach you how to Decrypt it in the Decrypt Site.

The final type of Encryption is SPC-Util Encryption. Drag your RSB/OBB to Taiji, Option number "08" will help you to encrypt it entirely.

How to make a new PAM by Twinkles

Use Twinkles to decode PAM file

Drag PAM file to twinkles and choose “PopCap-Animation decode” to convert the PAM to PAM.json, then drag PAM.json choose “PopCap-Animation JSON to Flash” to convert to Xfl floder

Next step, if u want use 768 textures to motify then drag Xfl flodern to Twinkles choose 33 and type 768, it will convert xfl can read 768 textures. If 1536 then type 1536

Add textures to PAM

Open library, create a media folder and copy all sprites of the pam here, open each source_x.xml and rename same name sprites in media folder

If ur sprites more than source files, just copy new source and image and + 1 for them. Ur sprites should suitable for game like size doesn’t so big or small

Ex: soruce_23 want to add new, copy source_23 rename to source_24, do same again image_x.xml

Modify the DomDocument, u add or remove sprites same all sprites was added

Make PAM animation

All u done, open main.xfl by Adobe Animate. It should be noted that the XFL animations that can be processed by Twinkles currently have many limitations, such as: only PNG bitmaps, no gradients, and only frame-by-frame production (traditional complementary frames can be used, and transcoding is required when Converted to frame-by-frame animation in Animate before). Refer to the following for specific instructions. If the following rules are violated, there is a high probability that the conversion of XFL to PAM will not be successful (the program will crash when doing the conversation)

Animation main structure description The main scene of the animation is named "animation", and it has 3 Layers, from top to bottom:

*1 flow: Label layer, used to set frame labels, the game program will locate the animation to be played through the frame labels and stop commands * stop instruction: the instruction stop() , which will stop the playback of the animation..

*2 actions : define action commands Through the cooperation of the frame tag and stop, the game can play the animation between the two. Other action commands: fscommand command, the game can read this command and make the object perform the specified behavior: for example: fscommand("use_action"); command, which can make the shooter plant shoot a bullet and make the production plant Produce a sunlight. * audio : fscommand command, used to play sound effects, such as: fscommand("PlaySample", eventName); , where eventName is the string ID of the Event defined in SoundBank

3* sprite : animation layer, this layer directly calls the "movie clip" component whose ID is "main_sprite". To modify the animation content, you should directly step into the modified component for modification, and do not change the data of this layer

Open library double click to main_sprite and u can see a lot layers then delete all layers if not u could crash when encoding to PAM

Create new layer, drag a image sprite for only one layer and u should take sprites in image folder if don’t make animate for zombies or switching attack plants. Take sprite in sprite folder for a component's label can be used for . When the game is playing the animation, the Sprite elements that specify certain labels are not displayed. The "plant dress-up switching" in the game is this principle

A sprite can have the following properties: Be * Pan, zoom, rotate, tilt, alpha. Attributes other than these five attributes will be ignored in the process of encoding PAM. “Do Not take more than 1 sprite in a layer if not pam will be crash when u encode PAM again”

Transcoding is required when Converted to frame-by-frame animation u can use montion tween, classic tween to make animation then convert all frame to frame-by-frame animation

Modify extra.json file to suitable sprites

Open the extra.json file in the xfl folder, add a new member after the last member of the "ImageSize" object, the member Key is the image ID, corresponding to the "ID" in resources.json, the image ID is only uppercase It consists of letters, arrays, and underscores; the member value is an array consisting of two unsigned integers (without decimal points), which respectively represent the width and height of the image

Ex: raiden001 represents an image with ID IMAGE_PLANT_RAIDEN_RAIDEN001

“size” is width 250 pixels and height 140 pixels

Add each sprite to image. In W and H, the scaling ratio for >1536 resolutions is 78.125 % (percentage). So if you import a bitmap whose standard size is 100x100, the size of the actual image file should be 100x100 times 1/0.78165 (about 1.28) = 128x128, the scaling ratio for >768 resolutions is 128 %.

In sprite for switching event “plant dress-up switching”, u can pass this if u didn’t use sprite when u was making PAM

Encode PAM

Drag xfl folder to Twinkles and choose PopCap-Animation JSON from Flash, then drag PAM.json choose PopCap-Animation encode. In case PAM fails when encoding, u need to check the PAM rules and correct them

Try to open PAM.ripe.json by Animate Viewer if it can view, u made it correctly and it has 95% success

DISCLAIMER: Guide is by Hky, big thanks to Vitaliy100 for additional

First, you need an app called APK Studio (make sure you install the required binaries correctly), as well as an app called APK Editor

Let's say you want to change pvz2_row to pvz2_abc. (It can be any 3 letters)

Use APK Studio to unpack the APK (leave the default checkboxes). It unpacks the APK contents into a folder with the same name and location as your APK file.

Now open the folder, go to smali_classes2/com/sexyactioncool/lawn/BuildConfig.smali

In that file change "pvz2_row" to "pvz2_abc", also change the "row" to "abc". Don't change the one that says "rowThin"

Now go back to APK Studio to repack the APK (click the hammer on the bottom left).

Now, inside the unpacked APK folder (the same folder that contained the smali_classes2 from above), you should see a new folder called "dist". Open it and you should find the new packed APK inside.

Updated by Vitaliy100: With 9.7+ you need to work with the classes3 and classes4

For me, the APK this produces is invalid. However, there is one part of the output we want to use.

Open the APK in the "dist" folder with an archive manager like 7-Zip, then copy the classes2.dex file. We will need it later!

Download APK Editor Studio and open the vanilla APK (not the one produced by APK Studio). Then click the "Open Contents" button in the middle to open the folder

Now do the following changes:

  • Open AndroidManifest.xml, find the line with GooglePlayExpansionEnabled and change it from true to false
  • Also in AndroidManifest.xml, replace all occurrences of pvz2_row to pvz2_abc
  • Copy the classes2.dex file from earlier into the APK folder, replacing the one that is already there
  • Open lib/arm64-v8a/libPVZ2.so with a hex editor, search for pvz2_row and replace with pvz2_abc
  • Do the same in lib/armeabi-v7a/libPVZ2.so (if using HxD delete the libPVZ2.so.bak file produced, otherwise it will make the final APK unnecessaily big)
  • Click Save APK. You've got the Custom APK! (Note: if you want to change the name of the app, go in res/values/strings.xml, then change app_label and app_name)

Install this APK on your device or emulator

Rename your OBB file replacing pvz2_row with pvz2_abc. No other changes to the OBB are needed

Create a folder on your device (Android/obb/com.ea.game.pvz2_abc). (Or you can open the app, it will instantly crash but create this folder for you)

Paste your obb in this folder, it should be "main.563.com.ea.game.pvz2_abc.obb" (the three numbers can be different depending on your version)

Now run the apk and it should work

Guide to force the game to only use one type of texture (384/768/1536)

To do that you have to edit the lib file of the game in apk, which located in {libdirectory}/lib/arm64-v8a/libPVZ2.so and {libdirectory}/lib/armeabi-v7a/libPVZ2.so

The replace all hex value of other types of texture to the type of texture you want.

384: 80 01 00 00

768: 00 03 00 00

1536: 00 06 00 00

Then press Ctrl + S to save the file

Do for both libPVZ2.so file in both folder (arm64-v8a & armeabi-v7a).
