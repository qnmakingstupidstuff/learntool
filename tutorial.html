I will explain how the Encryption Method works.

The first type of RSGP or PGSR is called "Uncompressed". This type of Format is Popular since 1.4.2 to 3.1 but after that PopCap & EA switched into the type "Compressed". This isn't a popular type of RSGP anymore but they are still using it. One common is "StreamingWave" and "__MANIFESTGROUP__". PvZ2 Chinese still use this type of RSGP, however.

The second type of RSGP or PGSR is called "Compressed". This is the most popular structure in the RSB or OBB. If "__MANIFESTGROUP__" from "Uncompressed" to "Compressed", the game will still run normally but not include for "StreamingWave".

Infact, Encryption is very very easy. They just used the "bug" called "corruption". They will "corrupted" the tools which read header of the rsgp and see the "01" or "03" to handle the RSGP. So, they just change the number "01" or "03" to something similar to "00" or "FF"

The next type of Encryption is the RSB Read Encryption. Although this Encryption method is old, people still like it and use it. Take example from the SHUTTLE OBB by CZ

So with this encryption, tool like Taiji, SPC-Util or TwinKleS when read will jump into the Offset "FFFFFFFF" and found nothing because that Offset doesn't existed. In order to do this kind of encryption, make a blank RSGP with anything and change the offset to "FFFFFFFFF". With this way, you are encrypting the OBB from powerful tool. However, i will teach you how to Decrypt it in the Decrypt Site.

The final type of Encryption is SPC-Util Encryption. Drag your RSB/OBB to Taiji, Option number "08" will help you to encrypt it entirely.

FORMAT

SFX - Replace 37 BC B7 E2 00 00 00 00 with 00 00 00 00 E9 58 17 2F

Music - Replace 37 BC B7 E2 00 00 00 00 with 00 00 00 00 37 53 E0 14

Both - Replace C3 00 00 01 with C0 00 00 01 (ONLY those right of above bytes)

Adding SFX

Prepping the audio file

1. Create a wav file of whatever SFX you want. (Can be mono or stereo, doesn't matter)

2. Open Wwise (the version PvZ2 uses) and open a new project.

3. Project (Top Left) -> Import Audio Files -> Add Files, then add all the wem files you want to play, then click Import. Sounds will be found in Project Explorer -> Audio -> Actor-Mixer Hierarchy -> Default Work Unit

4. Right click the Default Work Unit above the wem file -> New Child -> Actor-Mixer. Add all the sound files under that Actor-Mixer

5. Right click on the sounds and click Edit to open Property Editor and change the Mode to Define Custom. You can edit volume in General Settings as well.

Change format to Vorbis.

If you want the SFX to loop you can set it to loop in General Settings.

6. Add a Play sound event for the SFX.

Right Click on sounds -> New Event -> Play

Give the event a name (preferably play_something). Remember this event name - You need it to play the SFX. (Eg: Play_Haunter_ChainRustle)

Alternatively, you can add an event directly from

Project Explorer -> Events -> Default Work Unit

7. Layout (top left again) -> Soundbank

8. Go Project Explorer -> Soundbanks -> Default Work Unit

Right-click Default Work Unit -> New Child -> Soundbank

What you name this will become the name of the bnk file (remember this too, you will add it into structinfo and resources).

9. Select the new Soundbank in Soundbank Manager, then drag Event over to the bottom right box (You can drag the parent or a folder the event is in instead)

Go back to Audio and drag the audio files (or parent) to that box too.

10. Tick the Soundbanks / Platforms / Languages then click Generate. You have generated the soundbank!

11. Find the generated bnk

Project -> File Manager, right-click the project folder link and click Open Containing Folder

Go to the file GeneratedSoundBanks -> Windows and copy the bnk file you created (NOT init.bnk)

12. Add the bnk into the obb

Find an existing rsgp audio file, copy it and rename it to whatever you want (remember the name). Unpack it and replace the bnk inside it with the one you created.

Edit the rsgp's StructInfo as well

Making volume slider work with the SFX/p>

13. Open the bnk with a hex editor.

14. Replace all occurrences of the following bytes

For SFX Slider

Replace 37 BC B7 E2 00 00 00 00 with 00 00 00 00 E9 58 17 2F

For Music Slider

Replace 37 BC B7 E2 00 00 00 00 with 00 00 00 00 37 53 E0 14

For Both

Replace C3 00 00 01 with C0 00 00 01 (Located to the right of above bytes)

Only replace the ones a few bytes to the right of 37 BC B7 E2 00 00 00 00!

(The first 8 bytes are for the game's volume slider. The last 4 bytes allow multiple of the sound to be played at once.)

15. Save the hex-edited bnk file and pack the rsgp file (the opened bnk is useless)

16. Regular obb editing stuff: Add the new rsgp file you created in the obb's structinfo and resources

Playing the SFX file

Firstly whatever that's playing the SFX file must also load that file. (eg for Zombies, add the AudioGroups into ZombieTypes)

Anything with animation can play whatever SFX. (You'll need to edit the animation)

In addition, anything with an editable sound can have its sound replaced with the one you made. (Search Play_)

Do note that in a level, objects past x=98 might not be able to affect sound.

(Here the SFX with the Play_Haunter_ChainRustle event will be played in frame 90 of the zombie's animation)

Wherever in the game's code that asks for an event, the event you made can also be used too.

Basically, wherever in the code where the game has Play_something you can make it play your custom audio instead.

(Eg: Projectiles have a ImpactSoundEvent parameter, birthday song module takes in a play event (the stop event doesn't work))

Adding Music

Note: Making the game play the music like normal tracks in game (changes every flag and after CYS) is difficult (aka idk how, but there's a workaround in the next section)

Music added using this will only be playable the same way as the sfx above. (Most likely through the Birthday Song Module, which lets you insert an event at the start of the level, and when the reward appears)

Adding music is the same as adding SFX, except these steps:

3. Import your tracks to Project Explorer -> Audio -> Interactive Music Hierarchy -> Default Work Unit -> Music Playlist Container -> Music Segment (Create child for playlist container and music segment)

3.1. Change Layout -> Interactive Music

Click on the Music Segment, then adjust the Music Segment Editor such that:

Entry Cue (hold CTRL to move) is where the music starts playing (Just keep this at the start)

Exit Cue is where the music loops (the current track will still play, but the 2nd loop track will start playing on top)

3.2. Click on the Music Playlist Container, and look at the Music Playlist Editor

Drag the Music Segment into the box, then change the Loop Count of the playlist (the top one) to infinite.

The rest of the steps are the same as editing SFX

Adding Custom Wave Music

Note: This is NOT how PvZ2 normally does wave music. This is a work-around because we can't use the actual switches popcap uses (at least not without a lot of hex editing), hence the switches do not automatically change, nor do each level save the existing state.

This method has these limitations:

1. You need to change each music manually through an event constantly, if you want to prevent wrong music from playing (this requires spawning stuff every flag wave, so this won't work in endless)

2. Gimmicks that pause music (piano, boombox) will stop the music permanently (includes jam - music doesn't work with EightiesStageProperties at all)

3. Music for flag waves begins after the wave starts, not when the "A huge wave of zombies is approaching" message appears.

4. No sound for the huge wave text appearing.

However, an advantage of this is more control over the music: You can change the music as long as you can trigger an event. This means you can change the music at any wave, when a specific zombie appears, when a trap tile is activated, etc

For this, the Birthday Song override module will be used. (the stop event does nothing fyi)

FYI: States are used rather than just Stop/Start events is because upon a restart, changes from Stop/Start events won't save - the CYS music will play.

Changes from states won't save either, but you can constantly correct a wrong state. Whereas through Stop/Start correcting it will reset the music.
